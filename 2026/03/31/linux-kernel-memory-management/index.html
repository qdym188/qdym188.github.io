一、物理内存 vs 虚拟内存

核心概念对比

特性 物理内存 虚拟内存
本质 真实的 RAM 硬件 进程看到的抽象地址空间
地址范围 0 ~ 物理内存大小 0 ~ 2^64（64位系统）
隔离性 全局共享，无隔离 每个进程独立，互相隔离
连续性 必须连续分配 可以非连续（通过页表映射）
大小限制 受硬件限制 可远大于物理内存（交换）

二、页表（Page Table）

为什么需要页表？

实现虚拟地址 → 物理地址的转换，同时提供权限控制（读/写/执行）。
多级页表（以 64位 ARM64 为例）

地址转换流程:

CPU 发出虚拟地址 0x0000_1234_5678_9ABC

  1. 取 PGD 索引 (位[47:39]) = 0x02
    PGD[2] → PUD 基地址

  2. 取 PUD 索引 (位[38:30]) = 0x11
    PUD[0x11] → PMD 基地址

  3. 取 PMD 索引 (位[29:21]) = 0x0A
    PMD[0x0A] → PTE 基地址

  4. 取 PTE 索引 (位[20:12]) = 0x345
    PTE[0x345] → 物理页框基地址 0x8000_0000_0000

  5. 取 Offset (位[11:0]) = 0x9ABC
    物理地址 = 0x8000_0000_0000 + 0x9ABC = 0x8000_0000_9ABC

malloc 分配策略

分配大小 实现方式 系统调用 释放方式
< 128KB 堆分配 brk() 不立即归还，缓存复用
>= 128KB 匿名映射 mmap() munmap() 立即归还
超大内存 文件映射 mmap(文件) munmap()
