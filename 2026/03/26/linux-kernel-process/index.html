Linux Kernel 的进程管理是其最核心的子系统之一。本文主要梳理fork/exec、调度器(CFS)、进程状态转换的概念和使用方法

进程管理在 Kernel 中的位置

Fork：进程的”自我复制”

一句话定义： fork() = 细胞分裂。一个进程变成两个几乎完全相同的进程，父子共享代码，独立运行。

CFS是什么

CFS（Completely Fair Scheduler，完全公平调度器） = Linux 的默认进程调度器，自 2.6.23（2007年）取代 O(1) 调度器，追求公平性而非绝对效率。

核心思想：

CFS数据结构

1. 调度实体（sched_entity）

2. CFS 运行队列（cfs_rq）

CFS核心机制：红黑树 + vruntime

vruntime 计算详解

  • nice 值小的进程（高优先级），vruntime 增长慢 → 更容易被选中
  • nice 值大的进程（低优先级），vruntime 增长快 → 等待时间更长

CFS 与实时调度器对比

特性 CFS（SCHED_NORMAL） 实时调度器（SCHED_FIFO/RR）
目标 公平分享 CPU 绝对优先级
算法 红黑树 + vruntime 优先级队列
抢占 自愿（时间片用完） 强制（高优先级立即抢占）
nice/优先级 nice -20~+19 1~99（实时优先级）
适用场景 普通进程 实时任务（音视频、控制）

CFS 的局限与演进

局限

  1. 对延迟敏感任务不友好（vruntime 累积延迟）
  2. 多核负载均衡开销大
  3. 无法感知业务优先级（仅 nice 值）

演进：Sched Ext（BPF 调度器）
